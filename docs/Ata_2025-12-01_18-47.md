#### **1. Objetivo da Sprint**

Alpha iniciou a reunião definindo o objetivo duplo da Sprint 7:
1.  **Persistência de Dados:** Garantir que o banco de dados da aplicação seja resiliente e sobreviva a reinicializações do container, desacoplando os dados da aplicação.
2.  **Refinamento de UX:** Elevar a qualidade da interação do usuário implementando uma funcionalidade de edição de itens intuitiva (Long Press) e aprimorando a coesão visual da interface.

---

#### **2. Planejamento e Diretrizes (Visão do Architect)**

O Architect detalhou a visão técnica para alcançar os objetivos, dividindo as tarefas em duas fases principais:

*   **Fase 1 (Backend & Persistência):**
    *   **Estrutura:** Mover o banco de dados SQLite para uma pasta `data/` na raiz do projeto.
    *   **API:** Criar um novo endpoint `POST /update_item` para edição de itens.
    *   **Schema:** Modificar o script `reset_db.py` para criar a pasta `data/` e popular o banco com dois usuários fixos (`thiago` e `debora`), removendo o usuário `admin`.

*   **Fase 2 (Frontend & Docker):**
    *   **Interação:** Implementar um evento de Long Press (600ms) nos cards dos itens para acionar a edição.
    *   **UI:** Desenvolver um modal de edição com estética *Glassmorphism/Dark Neon*.
    *   **Estilo:** Ajustar o CSS da classe `.user-badge` para uma aparência mais suave e integrada ao design.
    *   **Infraestrutura:** Configurar o `docker-compose.yml` com um volume (`./data:/app/data`) para garantir a persistência do banco de dados no host.
